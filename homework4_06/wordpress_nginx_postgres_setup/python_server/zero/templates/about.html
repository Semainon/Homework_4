Описание проекта

Введение

Данный проект представляет собой комплексное решение для развертывания веб-приложений с использованием контейнеризации и современных технологий. Основные компоненты включают в себя Nginx, PostgreSQL, Django, WordPress, а также инструменты для мониторинга и CI/CD.

Технологическая архитектура

Проект включает следующие ключевые компоненты:

  • Nginx: Используется в качестве HTTP-интерфейса для WordPress.
  • PostgreSQL: Реляционная база данных, которая хранит данные WordPress и предоставляет возможности для масштабирования.
  • WordPress: Cистема управления контентом, используемая для создания и управления веб-сайтами.

Контейнеризация с Docker

Для развертывания всех компонентов используется Docker. Основные файлы конфигурации включают:

  • Dockerfile: Определяет процесс сборки образов для Nginx, PostgreSQL, Django и WordPress.
  • docker-compose.yaml: Обеспечивает координацию между контейнерами, включая настройку переменных окружения для суперпользователя PostgreSQL и конфигурацию WordPress.

Мониторинг с Grafana

В проекте также реализован мониторинг с использованием Grafana. Данные о метриках PostgreSQL передаются в Grafana для визуализации и анализа производительности системы.

Контейнеризация веб-сервера

В рамках проекта был контейнеризован веб-сервер на Python/Django.

Модернизация конфигурации

Конфигурация Nginx была модернизирована для хранения логов и конфигурационных файлов вне Docker volume, что обеспечивает более гибкое управление и доступ к логам.

При обращении к HTTP-пути /myWebBack открывается страница сервера, реализованного на Python/Django.

CI/CD Pipeline

В проекте реализован CI/CD pipeline, который:

  • Использует переменные для управления конфигурацией (имена пользователей СУБД, пароли, номера портов и т.д.).
  • Тестирует сборку контейнера и его работу, включая доступность Grafana, работу WordPress и сервера на Python.
  • Доставляет собранный контейнер на конечную машину и запускает его.
